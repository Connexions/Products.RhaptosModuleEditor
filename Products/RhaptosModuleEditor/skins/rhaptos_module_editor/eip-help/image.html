Description

The image tag is used to place image files, such as JPG, GIF, and PNG, into modules. Any image file with a valid MIME type can be included on the page.

Example

This CNXML:


<media id="greyhound" 
       alt="A greyhound relaxing on a bed.">
  <image mime-type="image/jpeg" 
         src="image1.jpg" />
</media>

Example

This CNXML:


<media id="greyhound" 
       alt="A greyhound relaxing on a bed.">
  <image mime-type="image/jpeg" 
         src="image1.jpg" />
  <image mime-type="application/postscript" 
         src="image1.eps" 
         for="pdf"/>
</media>

Results in this display:

This example demonstrates the use of the for attribute to specify an alternate image for the PDF version of the module. The Web-ready JPEG image, here used as the default option, will be displayed when viewing the module online and the second image, a higher-resolution EPS file, will be used for the PDF version.

Attributes(what's this?)

  • src (required): The location of the image file. It must be a URL value. If the image is stored alongside the module, the file name itself is sufficient (e.g. "img.jpg").

    Note:

    Starting with version 0.6, the CNXML language does not permit URLs that contain ampersand (&) characters (i.e., http://cnx.org/example?i=1&p=2). Check all object src URLs and replace any ampersands with the entity reference "&amp;". More information can be found on the W3C's page.
  • mime-type (required): The MIME type of the image. This requried value tells each visitor's browser what sort of information is contained in the image. The most commonly used image files and their corresponding MIME types are: "image/jpeg" (for .jpg images), "image/gif" (for .gif images), and "image/png" (for .png images). A list of common Internet media MIME types can be found here.
  • for (optional): Specifies whether this object should be displayed in the online or PDF version of the content, or be used as the default media object. If there is no default media object present, then the <media> element will only be displayed for the version(s) specified.
    Possible values:
    • default - This element is used for both PDF and online versions unless another, version-specific element is present (default).
    • pdf - This element is used only for the PDF version of the content.
    • online - This element is used only for the online version of the content.
  • height (optional): The height of the image in pixels. Must be a nonnegative integer.
  • width (optional): The width of the image in pixels. Must be a nonnegative integer.
  • print-width (optional): The width of the image for the PDF version of the module. Must be specified as a number plus a unit of width (e.g., "5in"). Supported units include "in" (inches), "cm" (centimeters), "pt" (points) and others. There is no corresponding "print-height" attribute. The height of the image in PDF will be determined proportionally.
  • thumbnail (optional): The URL of an image file to be used as a thumbnail for the larger image. The thumbnail file will automatically link to the larger image file set in the "src" attribute.
  • longdesc (optional): The URL of a file containing a long description of the image. Not all readers will be able to see your included media in their browser, either due to blindness, low-vision, or an environment that doesn't support the media. Some images are too complex to describe fully in a few words using the "alt" attribute, however. For images like charts and graphs that are important for understanding the material, use "longdesc" to point to a Web file that contains a full explanation of the image.
  • id (optional): A unique identifier, whose value must begin with a letter and contain only letters, numbers, hyphens, underscores, colons, and/or periods (no spaces).
